# Data Dictionary

This document describes the main fields included in the EEG dataset and associated metadata files.

The primary dataset is hosted on Zenodo:

- Concept DOI: https://doi.org/10.5281/zenodo.20089289
- Version 1.0.0 DOI: https://doi.org/10.5281/zenodo.20089290

## Raw EEG CSV files

The raw EEG files contain electroencephalographic recordings acquired using Muse 2 wearable dry-electrode EEG headsets.

| Field name | Data type | Units | Description | Notes |
|---|---:|---|---|---|
| `timestamps` | numeric or datetime | seconds or ISO timestamp | Time reference for each EEG sample. | The format depends on the acquisition software. If numeric, specify whether it represents relative time or Unix time before analysis. |
| `TP9` | numeric | microvolts (µV) | Raw EEG signal from the left temporal electrode TP9. | Do not apply filtering unless the file is explicitly labeled as processed. |
| `AF7` | numeric | microvolts (µV) | Raw EEG signal from the left frontal electrode AF7. | Muse 2 frontal channel. |
| `AF8` | numeric | microvolts (µV) | Raw EEG signal from the right frontal electrode AF8. | Muse 2 frontal channel. |
| `TP10` | numeric | microvolts (µV) | Raw EEG signal from the right temporal electrode TP10. | Muse 2 temporal channel. |
| `Marker0` | numeric or categorical | none | Temporal marker associated with experimental segments or cognitive tasks. | Marker values should be interpreted according to the experimental protocol. |
| `Right AUX` | numeric | microvolts (µV) | Auxiliary channel, if present in the acquisition file. | This field may be absent in some files. |
| `Elements` | text or numeric | none | Additional field generated by the acquisition software, if present. | This field may not be required for EEG analysis. |

## EEG channels

| Channel | Location | Description |
|---|---|---|
| `TP9` | Left temporal | Dry-electrode EEG channel from the Muse 2 headset. |
| `AF7` | Left frontal | Dry-electrode EEG channel from the Muse 2 headset. |
| `AF8` | Right frontal | Dry-electrode EEG channel from the Muse 2 headset. |
| `TP10` | Right temporal | Dry-electrode EEG channel from the Muse 2 headset. |

## Sampling frequency

| Field | Value | Notes |
|---|---:|---|
| Nominal sampling frequency | 256 Hz | Researchers should verify timestamp consistency before analysis. |

## Experimental groups

| Code | Meaning | Description |
|---|---|---|
| `CONTROL` | Control group | Participants assigned to the control condition. |
| `EXP` | Experimental group | Participants assigned to the experimental condition involving ASMR-based audiovisual stimulation. |

Depending on the filename convention, the experimental group may appear as `EXP`.

## Cognitive-task information

The experimental protocol involved audiovisual materials containing different types of information:

| Code or label | Meaning | Description |
|---|---|---|
| `R` | Relevant information | Segment associated with relevant information presented to the participant. |
| `I` | Irrelevant information | Segment associated with irrelevant information presented to the participant. |
| `F` | False information | Segment associated with false information presented to the participant. |

Some analyses may use more specific labels such as `I1`, `I2`, `F1`, `F2`, `R1`, `R2`, or `R3`, depending on the segmentation procedure.

## Subject-level metadata

The file `metadata_subjects.csv` contains anonymized subject-level metadata.

| Field name | Data type | Allowed values or format | Description | Notes |
|---|---:|---|---|---|
| `subject_id` | text | `sub-001`, `sub-002`, ... | Anonymous subject identifier. | Does not contain names, student IDs, or institutional identifiers. |
| `original_file` | text | filename | Original EEG CSV filename. | Used for traceability between metadata and raw EEG files. |
| `group` | categorical | `control`, `experimental` | Experimental group assignment. | The experimental group corresponds to ASMR-based audiovisual stimulation. |
| `sex_code` | categorical | `H`, `M` | Anonymized demographic sex code retained for group-level analysis. | H/M follows the study coding convention. |
| `device` | text | `Muse 2` | EEG acquisition device. | Wearable dry-electrode EEG headset. |
| `sampling_rate_hz` | numeric | `256` | Nominal sampling frequency in Hz. | Verify timestamps before analysis. |
| `channels` | text | `TP9;AF7;AF8;TP10` | EEG channels included in the recording. | Channel names follow Muse 2 convention. |
| `acquisition_datetime` | datetime or text | `YYYY-MM-DD HH:MM:SS` or filename-derived format | Date and time of acquisition, when available. | May be derived from the original filename. |
| `dataset_version` | text | `1.0.0` | Dataset version. | Corresponds to Zenodo version 1.0.0. |
| `notes` | text | free text | Additional comments about the recording. | Optional. |

## Filename convention

Some files may follow a filename convention similar to:

```text
EEG_01_H_EXP_2024-10-31-12_25_25.csv

```

A filename of this type can be interpreted as:

| Segment | Example | Meaning |
|---|---|---|
| Prefix | `EEG` | EEG recording file. |
| Subject number | `01` | Internal subject number. |
| Sex code | `H` | H/M demographic code. |
| Group | `EXP` | Experimental group. |
| Date and time | `2024-10-31-12_25_25` | Acquisition date and time. |

For public documentation and analysis, researchers should use the anonymized `subject_id` field rather than relying only on the original filename.

## Marker interpretation

The `Marker0` field contains temporal markers associated with the experimental protocol.

A general interpretation is:

| Marker value | Meaning |
|---|---|
| `0` | Baseline or no active segment, depending on the acquisition protocol. |
| nonzero value | Event or experimental segment marker. |
| missing/NaN | No marker or inactive marker field. |

Researchers should inspect the marker distribution before segmentation.

## Recommended preprocessing notes

Before using the dataset for machine learning or statistical analysis, researchers should document preprocessing choices, including:

1. Timestamp verification.
2. Missing-value handling.
3. Artifact inspection or rejection.
4. Filtering parameters, if applied.
5. Segmentation strategy using `Marker0`.
6. Feature extraction method.
7. Train/test split strategy.

For machine learning experiments involving human participants, subject-independent validation is recommended when the goal is to evaluate generalization across subjects.

## Privacy and anonymization

Personal identifiers were removed or anonymized before publication. The dataset should not include names, student IDs, institutional IDs, email addresses, or direct personal identifiers.

The `sex_code` variable was retained as an anonymized demographic variable because it is relevant for group-level EEG analyses.
